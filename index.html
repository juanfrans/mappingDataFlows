MAPPING DATA FLOWS

Understanding how the largest technology companies collect, use, and share user information across the internet. We’ve transformed the “Big Four” terms of service and data policies -- the thousands of lines of code that govern their use of your data -- into a database powering an interactive visualization, an initial version of which we invite you to explore and critique.

On Facebook, personally identifiable information like Social Security Numbers, date of birth, and gender are regulated and protected. Users can control the app’s access to GPS location, cameras, and contact lists.

>

However, even with the most restrictive privacy settings enabled, Facebook can collect logs of the websites and videos users interact with, cursor movements, identifiers for electronic devices, and information about the cell phone towers and internet networks users are connected with.

>

Even with GPS location disabled, Facebook can pinpoint user locations and learn more about personal behavior than any information a user profile could provide. Furthermore, unlike PII, Facebook is free to share device and behavior information with third parties for any purpose.

The data behind this visualization was created base on the following terms of service and privacy policies:

Apple Privacy Policy, last accessed on October 3, 2019.

Facebook Data Policy, last accessed on October 3, 2019.

Amazon Privacy Notice, last accessed on October 3, 2019.

Google Privacy & Terms, last accessed on October 3, 2019.

However, note that there are many more terms of service and privacy policy documents for each of these companies. Due to time constraints we chose to focus only on the main document for each one of them.

This visualization is generated using three csv files:

Nodes.csv, which generates the data source, data type, and purpose nodes.

Generates.csv, which links the data source and the data type nodes.

CollectsUsesShares.csv, which links the data type and the purpose nodes.
